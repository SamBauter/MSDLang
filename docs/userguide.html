User guide

Running From the Command Line

SUMMARY:

>./MSDLangExec [--modeOfOperation] [fileContainingExpression]

The simplest way to run MSD Script from the command line is to run the program without any arguments:

>Ex: ./MSDLangExec

By default the program will allow you to enter a valid expression according to the grammar of the program terminating with an eof. In order to terminate with an eof you can press enter and then ctrl+d. Once eof character is detected the MSDScript will interpret the entered expression if possible. Note this mode is best used for expressions with small amounts of recursion as expression with large amounts of recursion may trigger a segmentation fault. For expressions which require large amounts of recursion the "--step" flag is the appropriate solution.

Ex: 1+2

Result: 3D (Note that the D from ctrl + D will be displayed for single digit answers)

Ex: 1+20

Result: 21

Interpreting an Expression From a Text File

When running the program a file can be provided as the first argument. Without any other arguments provided the MDSScript will read and interpret the expression contained in the file.

./MSDLangExec [fileContainingExpression]

Ex: ./MSDLangExec ./factorial5.msd

>Result: 120

Optimizing an Expression

The "--opt" flag can be provided as the first argument of the command. This flag signals to the program will operating in optimize mode and attempt to simplify the passed expression(expressions can be passed in the command line or through files as before). For more information on the specific type of simplification that is performed on the program please refer to the Optimize section of the API documentation.

./MSDLangExec --opt [fileContainingExpression]

Interpreting Expression with a large amount of recursion

The "--step" flag is a flag that changes the way memory is allocated during the evaluation of an expression. Use this flag if the default mode of interpretation segmentation faulted. An example of the type of expression that step is designed for can be seen below.

Example Expression in file count.msd:

_let count = _fun(count)

_fun(n)

_if n == 0

_then 0

_else 1 + count(count)(n + -1)

_in count(count)(100000) Ex: ./MSDLangExec --opt ./count.msd

Result: 100000

Running Tests

The MSDLang Package contains more than 100 catch2 tests in a file Tests.cpp which is included in main. For those interest in using this language as a starting point and adapting it further tests can easily be added to that file. In order to run the tests the flag "--test" can be passed as the first argument in the command line.

Ex: ./MSDLangExec --test

Result: All tests passed (162 assertions in 67 test cases)
