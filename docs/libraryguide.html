Using MSDLang as a Library

Some users may wish to use MSDLang as a Library. Doing so is relatively simple. Perhaps the most basic example of library usage is a developer that wishes to create a program that will prompt the user for input and run that input through a function they have designed in MSDLang. In order to accomplish such a task. The developer will need to create a main file that contains or references the string representation of their MSDLang function, and will need to include all of the necessary header files for the functions the MSDLang Library functions they intent to use.

Once the main file has been created and the correct header files have been included, the main file can be compiled and linked with the library file. An example of compiling and linking these files is shown below:

Example of Compiling a main and linking with the library file: clang++ --std=c++11 counting_user.cpp libmsdscript.a

Example source code that shows using MSDScript as a library to design a program that counts to a certain number for a user can be found here

A common pattern for interpreting expressions in a string representation will be to use variants of the following functions in this order:

  1. Begin by using parse on the expression to produce an Expr Object. For more information on parse you can refer to either the Grammar Guide photo which spells out parsing hierarchy. Alternatively, the function signatures for parse can be found in the ExprParser.hpp file.
  2. Interpret that expression. Different interpret methods require different parameters the example code linked above shows both standard interp using and Env and an alternative method using the interp_by_steps function. These functions can be found in the in the Expr.h or Step.hpp file respectively.
  3. Printing the result of the interpreted expression using a to_string method both Expr and Val classes contain this method.
